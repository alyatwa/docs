---
title: 'Benchwrk Logger'
description: 'Send logs to Benchwrk Chrome extension using @benchwrk/logger'
---

<Info>
  **Prerequisites**:
  - Node.js version 22 or higher
  - Benchwrk Chrome extension installed
</Info>

Follow these steps to install and use the @benchwrk/logger package.

<Steps>
<Step title="Install the @benchwrk/logger package">

```bash
npm install @benchwrk/logger
```
</Step>

<Step title="Import and initialize the logger">

In your JavaScript or TypeScript file, import the Logger and initialize it with your API key:

```javascript
import { Logger } from "@benchwrk/logger";

// Initialize logger with your API key
const logger = new Logger({ apiKey: "12345" });
```
</Step>

<Step title="Log messages">

Use the logger to send logs to the Benchwrk Chrome extension:

```javascript
// Log messages with different levels
logger.info("User logged in successfully", { userId: 123 });
logger.warn("Performance degradation detected");
```
</Step>
</Steps>

## Supported log levels

- `info`: For general information messages
- `warn`: For warning messages
- `error`: For error messages
- `debug`: For debug information

## Additional options

You can pass additional metadata as the second parameter to any log method. This metadata will be included in the log sent to the Benchwrk extension.

Example:

```javascript
logger.error("Database connection failed", { errorCode: 500, userId: 123 });
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Logs not appearing in Benchwrk extension">
    Ensure the Benchwrk Chrome extension is installed and enabled. Check that the API key is correct and the extension is configured to receive logs from your application.
  </Accordion>

  <Accordion title="Installation issues">
    Make sure you have Node.js 14 or higher installed. Try clearing your npm cache: `npm cache clean --force` and reinstall.
  </Accordion>
</AccordionGroup>
